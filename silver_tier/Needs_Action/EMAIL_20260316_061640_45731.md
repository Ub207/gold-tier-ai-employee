## Summary



I built `agent-evidence`, an open-source package that exports agent execution

into verifiable evidence bundles with:



- JSON / CSV / XML export

- signed manifests

- multi-signature support

- key rotation

- threshold policies

- SQLite / PostgreSQL backends



## Why this may fit OpenAI Agents SDK



The SDK already exposes custom trace processors, so Agent Evidence can plug in

without changing the core runtime. I now have a working integration branch that

maps OpenAI Agents traces and spans into the same semantic evidence model used

for our existing LangChain support.



## Proposed contribution



I would like to contribute a minimal example or docs patch showing how to

export OpenAI Agents traces into Agent Evidence bundles via a custom trace

processor.



Concretely, the branch currently includes:



- an `AgentEvidenceTracingProcessor`

- a local example using `trace()` and `custom_span()`

- regression tests covering trace/span lifecycle capture
