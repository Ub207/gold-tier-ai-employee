## Problem

Fixes #1167



When using multiple MCP servers that provide tools with the same name (e.g., multiple filesystem servers), tool names collide and cause errors.



## Solution

Add `tool_name_prefix` parameter to MCPServer base class:



1. **Prefix tool names**: When `tool_name_prefix` is set, tool names are prefixed (e.g., `read_file` becomes `filesystem_read_file`)

2. **Strip prefix on call**: When calling a tool, the prefix is stripped to call the actual tool

3. **Backward compatible**: Default is `None`, no change to existing behavior



## Usage

```python

server1 = MCPServerStdio(

    params={&quot;command&quot;: &quot;mcp-server-filesystem&quot;, &quot;args&quot;: [&quot;/path&quot;]},

    tool_name_prefix=&quot;fs1&quot;  # tools become fs1_read_file, fs1_write_file, etc.

)



server2 = MCPServerStdio(

    params={&quot;command&quot;: &quot;mcp-server-filesystem&quot;, &quot;args&quot;: [&quot;/other&quot;]},

    tool_name_prefix=&quot;fs2&quot;  # tools become fs2_read_file, fs2_write_file, etc.

)

```
