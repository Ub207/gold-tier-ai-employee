### Describe the bug

`responses.compact` with **gpt-5.4** strips reasoning items from its output but retains assistant messages with their original IDs. When the compacted output is sent back to `responses.create`, the API rejects it because each assistant message ID implies a required paired reasoning item that was removed during compaction.

**This is gpt-5.4 specific.** The same workflow with gpt-5.2 works fine — gpt-5.2's `responses.compact` does not retain assistant message IDs in its output (0 assistant messages with IDs vs 3 for gpt-5.4).

This means **any conversation using reasoning + compaction with gpt-5.4 is permanently broken after the first compaction**.

`reasoning_item_id_policy="omit"` does not help because it strips IDs from reasoning items, not from the assistant messages that reference them.

### Debug information
- Agents SDK version: v0.12.5 (also reproduced on v0.10.5)
- Python version: Python 3.13.5
- OpenAI SDK version: 2.29.0
- Model: gpt-5.4 (gpt-5.2 works correctly)
