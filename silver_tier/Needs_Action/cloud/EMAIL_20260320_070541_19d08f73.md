  ### Why this looks like an SDK bug



  In _FunctionToolBatchExecutor, the executor creates the task first

  and registers it in task_states after:



  task = asyncio.create_task(self._run_single_tool(...))

  self.task_states[task] = _FunctionToolTaskState(...)



  Then _run_single_tool() immediately does a lookup using

  asyncio.current_task():



  current_task = asyncio.current_task()

  self.task_states[current_task].in_post_invoke_phase = False



  Under asyncio.eager_task_factory, the coroutine can start running

  before self.task_states[task] = ... executes, so that lookup can

  raise KeyError.



  Textual enables eager tasks like this:



  loop.set_task_factory(asyncio.eager_task_factory)
