Hi, I recently implemented GLM-4/GLM-5 support in another project ([khoj-ai/khoj#1275](https://github.com/khoj-ai/khoj/pull/1275)) and can share some learnings about `reasoning_content` handling.



**For Zhipu GLM models:**

- GLM-4/GLM-5 returns `reasoning_content` in the response, but it's optional for subsequent turns (no 400 error if omitted)

- However, including it improves response quality for complex reasoning tasks

- The key is to check if the model response contains `reasoning_content` and preserve it in the conversation history



**Implementation approach:**

```python

# When building assistant message for next turn

assistant_message = {

    "role": "assistant",

    "content": response.content,

}

if hasattr(response, "reasoning_content") and response.reasoning_content:

    assistant_message["reasoning_content"] = response.reasoning_content

```



**For Kimi models:**

- As noted in the issue, Kimi requires `reasoning_content` to be included in subsequent turns

- This is stricter than GLM's behavior



Happy to contribute a PR if this aligns with the project's roadmap. The milestone 0.14.x seems like a good target.
