seratch left a comment (openai/openai-agents-python#2671)



Thanks for sharing this. Regarding the user interruption pattern, it is indeed a common need, but this SDK does not yet support the pattern. One potential solution would be to halt the agent loop in the middle (perhaps either a turn of a model call or tool output) and insert the new user message before the final output of the ongoing agent run. We need to make sure if this works well for common use cases.



As for the compaction use case, queueing the upcoming user message within the SDK could be possible, but essentially the compaction needs to be done before proceeding with its succeeding requests.



I don't think max_turns=1 could fit for your need here, but if you find some creative ideas with it or anything else, sharing those with us would be appreciated as well.
