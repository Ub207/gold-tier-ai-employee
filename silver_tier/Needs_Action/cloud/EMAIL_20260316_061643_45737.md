Hi OpenAI Agents team 👋



The new Agents SDK does handoffs really elegantly, but like most agent frameworks the coordination state is ephemeral — restart the process and the task graph is gone. I wanted to share a complementary protocol that could address this for long-running pipelines.



**[GNAP](https://github.com/farol-team/gnap)** (Git-Native Agent Protocol) is an open RFC draft defining how AI agents coordinate through a shared git repo. Zero servers, zero databases — just 4 JSON files and standard git operations.



**Why this is relevant to the Agents SDK:**



The SDK already supports multi-agent handoffs via `handoff()` — GNAP could serve as the **durable task queue** underneath. Rather than building a new stateful orchestrator, a GNAP-backed runner would:



1. Serialize pending tasks into `.gnap/tasks/` before handoff

2. Receiving agent pulls, finds its task, works, pushes result

3. Any agent (Claude Code, Codex, custom) joins by reading `agents.json`

4. Full audit trail via git history



**Comparison:**



| | Server needed | DB needed | Any LLM | Audit trail |

|---|---|---|---|---|

| GNAP | ❌ | ❌ | ✅ | ✅ git history |

| Custom orchestrator | ✅ | ✅ | Depends | Custom |

| In-memory handoff | ❌ | ❌ | ✅ | ❌ |



This would be especially useful for async workflows where agents might run on separate machines or at different times.



Would the team be open to exploring a `GNAPBackend` runner adapter? Happy to prototype it.



Spec + quickstart: https://github.com/farol-team/gnap
