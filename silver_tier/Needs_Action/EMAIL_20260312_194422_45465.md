On Tuesday I told you about the gap between how most developers use AI (copy-paste from a chat window) and how agentic coding works (the AI runs inside your project).



Today I want to show you what that actually looks like, because it's hard to appreciate until you see the workflow.



Here's a simplified version of what you'd do in the course:



You open your terminal. You tell Claude Code: "Scaffold a Python CLI app called second_brain with Click, a notes command, and a SQLite backend."



A few seconds later, you have a project. Not a code snippet in a chat window. An actual project: package structure, pyproject.toml, CLI entry point, database module, the works. You run it. It works.



Now you want to add a feature. Instead of explaining your project to a chat window, you type:



"Read the codebase. Add a search command that does full-text search across all notes. Write tests first."



The agent reads your files. It sees your existing code, your database schema, your test structure. It writes the tests, then writes the code to make them pass, then runs the tests to confirm. If something fails, it reads the error and fixes it. You watch this happen in your terminal.



When you're satisfied, you tell it to commit and push. It writes a commit message based on what actually changed, creates a PR on GitHub, and you move on to the next feature.



That's one cycle. In two days, you'll run through it enough times that it becomes second nature.



Can you learn this on your own? Sure. The docs are public. YouTube has videos. You could piece it together over a few months.
